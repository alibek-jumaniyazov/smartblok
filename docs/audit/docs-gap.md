# docs-gap

## Domain summary
TZ (docs v1.0, 2026-07-09, Uzbek) vs actual code (working tree with large uncommitted refactor). Stack: NestJS 10 + Prisma 5 (SQLite dev / Postgres via docker-compose), React 18 + Vite + TanStack Query + Tailwind v4; 17 API controllers under apps/api/src/*/; 22 pages under apps/web/src/pages/.

ROLE MODEL (TZ ch.3): 4 roles stored as String (no DB enum). ADMIN = everything incl. /users CRUD and most DELETEs. ACCOUNTANT = sales/catalog/finance/reports/import, some deletes (orders, payments, products), no users. AGENT = orders/clients/payments only, scoped to own agentId (TZ documents scoping only on findAll). CASHIER = payments/expenses/kassa + separate CashierDashboard. Catalog GETs (/agents, /regions, /factories, /products, /vehicles) have no @Roles (any JWT) — still true in code. Auth = JWT bearer, 7d, payload {sub,username,role,name,agentId}, never reloaded from DB (still true). Frontend has NO route-level RBAC — only nav-menu filtering (still true, App.tsx Protected checks token only).

FUNCTIONAL MODULES — implemented status: Dashboard (implemented, but CHANGED: now @Roles ADMIN/ACCOUNTANT/AGENT — CASHIER 403s; AGENT gets scoped figures, company cash/factory/vehicle debt hidden); Orders (implemented + EXTENDED: ownership checks, status validation + vehicle check on create, credit-limit enforcement on create, soft-cancel delete, orderNo=max+1 with P2002 retry, new /orders/new full-page form replacing TZ's Drawer); Clients (implemented; findOne/update STILL have no agent ownership check; delete now blocked when history exists); Agents (implemented; GET still unguarded, leaks all agents' sales/profit/collected to any JWT); Factories/Products/Vehicles/Regions (implemented per TZ); Procurement matrix + prices/routes (implemented per TZ); Payments (implemented + CHANGED: AGENT limited to CLIENT payments for own clients); Kassa (implemented + CHANGED: manual tx requires active box, amount>0, USD rate; DELETE restricted to MANUAL source per RM-06); Debts (implemented + CHANGED: factory/vehicle advances now clamped, adds factoriesAdvance/vehiclesAdvance, |balance|>=1 filter); Expenses (implemented; UPDATE endpoint still missing (RM-15 open); NEW undocumented UZS-only cashbox rule); Reports/svod (implemented; still no date filter, no server Excel/PDF export); Users (implemented, unchanged: any-body, arbitrary role strings, hard delete — SEC-R4/RM-01 open); Import (implemented + CHANGED: single $transaction, kassa posting, replace also wipes expenses + PAYMENT/EXPENSE cash mirrors); Auth/Profile (implemented; NEW fail-fast requireJwtSecret >=16 chars replaces documented dev fallback; timing-safe DUMMY_HASH login).

BIGGEST DIVERGENCE: the TZ's core rule "only DELIVERED/COMPLETED orders count toward debt/sales/profit" was replaced system-wide by apps/api/src/common/recognition.ts: RECOGNIZED_ORDER = status != CANCELLED (debt recognized at order creation, per owner decision), applied in debts, clients, agents, factories, vehicles, dashboard, reports, orders services; payments tied to CANCELLED orders are excluded from balances. All TZ formulas, FR texts, and acceptance criteria AC-BIZ-01/02 are stale. Roadmap items delivered: RM-03 (orders/payments only), RM-06, RM-07, RM-08, RM-13 (create only). Still open: RM-01/02 (DTOs, token freshness), RM-03 for clients, RM-10/11 (migrate, Decimal), RM-15, RM-20/21/23. No new Prisma models (15 models, no enums, no pallet/zalog tracking).
