# frontend-pages

## Domain summary
React 18 + Vite + TypeScript SPA (apps/web), TanStack Query v5, axios, react-router v6, Tailwind (CSS-var theme), framer-motion, recharts, lucide icons. UI language Uzbek; number/date locale ru-RU. Shared kit: EntityTable (client-side search/pagination-12/CSV export/skeleton/empty), Drawer, Modal (unused), Field/Input/Select, MoneyInput, KpiCard, Badge, Toaster, CommandPalette (Ctrl+K nav-only), Layout+Sidebar with role-filtered nav (lib/nav.ts). API layer lib/api.ts: untyped axios wrappers (all any), Bearer token from localStorage, 401 hard-redirect. lib/format.ts: fmtUZS/fmtNum/fmtDate/fmtShort. lib/orderStatus.ts: client copy of 6-step order state machine.

Pages (22): Dashboard — role-branched KPIs (sales/profit/debts/kassa), sales-trend + funnel charts, agent-performance table; read-only; separate CashierDashboard. Orders — EntityTable of all orders, detail Drawer; mutations: advanceOrder, deleteOrder. NewOrder — 2-column order wizard (agent/client/factory/product/vehicle/qty/prices), client-side profit-margin preview and credit-limit projection; mutation: createOrder with client-supplied unit prices. Clients — list with delivered/paid/balance; create client Drawer. ClientDetail — client totals, orders + payments lists; mutation: createPayment(CLIENT) incl. USD-with-rate form. Agents — list with sales/profit; create agent+login user. AgentDetail — read-only KPIs, clients and orders lists. Factories — list with balance; create factory. FactoryDetail — totals, products, orders, payments; mutation: createPayment(FACTORY). Products — CRUD product (factory-bound, cost/sale price per m³); create/update/delete. Vehicles — list; create vehicle. VehicleDetail — transport-fee totals, trips, payments; mutation: createPayment(VEHICLE). Procurement — landed-cost matrix per region (factory price + logistics/truck m³, bonus), cheapest-source banner; read-only; default region hardcoded 'beruniy'. Payments — all payments ledger; create payment (3-type toggle CLIENT/FACTORY/VEHICLE), delete payment. Debts — 3 debt tables (clients/factories/vehicles) + totals KPIs; read-only. Expenses — expense list + category manager; create/delete expense, create category. Kassa — cashbox cards (UZS/USD/Click/Bank) with balances, transaction table; create/delete manual kassa tx. Reports — 'Svod': per-agent totals table + factory payments table; read-only. Users — user CRUD with roles ADMIN/ACCOUNTANT/AGENT/CASHIER. Import — Excel upload with replace-all toggle (default ON); mutation: importExcel. Login — username/password; hardcoded demo credentials incl. passwords rendered as click-to-fill buttons. Profile — update own info/password.

Patterns: every list page = PageHeader + EntityTable + Drawer-form; forms are useState<any> + spread set(); mutations mostly qc.invalidateQueries() (whole cache); queries destructure only data — no isError anywhere; deletes are one-click with zero confirmation (grep confirms no confirm dialogs in src). Server orders.findMany has no take/skip — full-table fetches. No print, PDF, xlsx, saved filters, date-range filters, sorting, or role route guards.
