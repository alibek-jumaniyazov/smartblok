## 1. Poydevor (barcha sahifalarni bir zumda yaxshilaydi)

### 1.1 `StatCard` boyitildi (`components/StatCard.tsx`)
- **Kattaroq qiymat:** `lg` = 27px/34px (avval 20px), `md` = 20px.
- **Yangi `size` prop:** `'lg'` (hero) yoki `'md'` (zich ikkilamchi plitka). `md`
  kam balandlik (minHeight 78), kam padding — bir qatorda 5 tasi bo'sh joysiz
  sig'adi.
- **Kartani to'ldiruvchi area-sparkline:** `sparkline` berilganda, kartaning to'liq
  enini egallaydigan gradient-to'ldirishli sparkline pastki qirg'oqqa yopishadi
  (`stretch` + `area`). Bu — keng kartadagi bo'sh joyni yo'qotadigan asosiy usul.
- **Semantik ink token'dan:** qiymat/sparkline rangi AntD token'idan olinadi
  (`in→colorSuccess`, `owedToUs→colorError`, `weOwe→colorWarning`,
  `neutral→colorPrimary`) — SVG `stroke` uchun ishonchli, tema-aware,
  `--sb-money-*` CSS var'lari bilan mos.
- **Ixtiyoriy `icon` prop:** yuqori-o'ngda tint'langan ikonka plitkasi (trend
  yo'q kartalar uchun).
- **Yaxshiroq hover:** `translateY(-2px)` + `--sb-shadow-e2`, brend chegara.

### 1.2 `Sparkline` boyitildi (`components/SmallAtoms.tsx`)
- `area?: boolean` — chiziq ostida yumshoq yuqoridan-pastga gradient.
- `stretch?: boolean` — SVG konteyner enining 100% ini egallaydi
  (`preserveAspectRatio="none"` + `vectorEffect="non-scaling-stroke"`).
- `useId()` bilan noyob gradient id (bir sahifada bir nechta sparkline).

### 1.3 Token va utility'lar (`theme.ts` o'zgarmadi, `design.css` kengaytirildi)
Yangi CSS'lar (`design.css`):
- Brend gradientlari: `--sb-brand-gradient`, `--sb-brand-gradient-soft`,
  `--sb-auth-bg` (login/landing/hero sirtlar uchun).
- `.sb-kpi-grid` — `repeat(auto-fit, minmax(210px, 1fr))`, gap 14: kartalar juda
  kengaymaydi, ko'proq ustun zichlashadi.
- `.sb-stat-strip` — zichroq ikkilamchi statlar qatori.
- `.sb-panel` / `.sb-panel__head` / `.sb-panel__title` / `.sb-panel__body` —
  qayta ishlatiladigan panel primitivi.
- `.sb-stack` / `.sb-overline` — vertikal ritm va bo'lim sarlavhasi.

**Qoida (kelajakdagi ish uchun):** kam kartali bo'lim keng cho'zilgan kartalar
o'rniga — TREND bo'lsa katta sparkline'li karta, TREND yo'q bo'lsa zich `md`
plitkalar qatori ishlatsin. Karta ICHIDAGI bo'sh joy "buzuq" ko'rinadi;
kartalar ORASIDAGI/YONIDAGI bo'sh joy toza layout ko'rinadi.

## 4. Landing — ommaviy SaaS sahifasi (`pages/Landing.tsx` + `pages/landing.css`)

SmartBlok DASTURINI boshqa gazoblok dillerlariga reklama qiladigan animatsiyali
ommaviy sahifa. Hammasi `.lp` ildizi ostida izolyatsiya qilingan (ilovaga
sizmaydi). buissnes_crm `LandingPage` uslubidan ilhomlangan, SmartBlok domeniga
moslashtirilgan.

**Bo'limlar:** sticky glass navbar + scroll-progress → hero (kursor-kuzatuvchi
glow, gradient sarlavha, count-up statlar 12/4/6/2, magnit CTA, chip'lar) →
Modullar (12 spotlight karta) → Qarz zanjiri (4-node oqim: Zavod→Diller→Agent→
Mijoz, animatsion punktir o'qlar + 3 daraja) → Ish jarayoni (6 order status) →
Dashboard ko'rinishi (3D-tilt mockup: KPI + area chart + status bar'lar) → Rollar
(4) → Xavfsizlik (ledger/audit/RBAC/valyuta) → CTA band → footer.

**Animatsiyalar (barchasi `prefers-reduced-motion` mos):** aurora blob'lar,
scroll-reveal (IntersectionObserver + data-delay stagger), count-up (rAF + ease-out
cubic), magnit tugma, 3D tilt, spotlight kartalar, marching-dash SVG o'qlar.

**Signature:** `--lp-ease: cubic-bezier(0.22, 1, 0.36, 1)`; brend feruza
(#0F766E/#14B8A6/#2DD4BF); qorong'i "control-room" fon.

Zanjir: **Landing (`/`) → «Kirish» → Login (`/login`) → Dashboard (`/app`)** —
end-to-end tekshirildi.

## 6. Kelajak uchun qoidalar (dizayn intizomini saqlash)

1. **Karta ichida bo'sh joy = buzuq.** Trend'li metrika → to'ldiruvchi
   sparkline; trend'siz → zich `md` plitka. Kam kartani cho'zib yubormang.
2. **Har bir raqam — eshik.** KPI/stat bosiladigan bo'lsin (`to=`).
3. **Semantik ink faqat token/`--sb-*` dan.** Hardcoded hex yo'q; pul rangi
   faqat MoneyCell/BalanceTag orqali.
4. **Bitta identity bloki:** har sahifada `PageHeader` — sarlavha o'lchami
   lotereyasi yo'q.
5. **Bo'sh joyni harakatga chorlovchi kontent bilan to'ldiring** (worklist,
   kompakt statlar), soxta bo'shliq bilan emas.
6. **Motion faxriy (chrome) — raqamlar animatsiya qilmaydi.** Barcha animatsiya
   `prefers-reduced-motion` ga bo'ysunadi.
